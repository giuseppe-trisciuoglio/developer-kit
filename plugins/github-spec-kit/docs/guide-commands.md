# GitHub Spec Kit Commands Guide

This guide documents all commands available in the GitHub Spec Kit Plugin.

---

## Overview

The GitHub Spec Kit Plugin provides specialized commands for GitHub specification and workflow optimization.

### Available Commands

- **Integration**: 1 command for checking GitHub integration
- **Optimization**: 1 command for optimizing GitHub workflows
- **Verification**: 1 command for verifying GitHub specifications

---

## Commands

### `/speckit.check-integration`

**Purpose**: Check integration with GitHub specifications and validate setup.

**Usage:**
```bash
/speckit.check-integration
```

**Common use cases:**
- Validating GitHub integration setup
- Checking specification compliance
- Troubleshooting integration issues

---

### `/speckit.optimize`

**Purpose**: Optimize GitHub workflows and specifications for better performance and maintainability.

**Usage:**
```bash
/speckit.optimize
```

**Common use cases:**
- Optimizing GitHub Actions workflows
- Improving workflow performance
- Streamlining GitHub configurations

---

### `/speckit.verify`

**Purpose**: Verify GitHub specifications against best practices and requirements.

**Usage:**
```bash
/speckit.verify
```

**Common use cases:**
- Verifying workflow specifications
- Checking compliance with standards
- Validating GitHub configurations

---

## Command Usage Guidelines

### How to Invoke Commands

Commands are invoked using the slash syntax in Claude Code:

```bash
/speckit.{command-name} [arguments]
```

### Best Practices

1. **Check integration first**: Use `/speckit.check-integration` to validate setup
2. **Optimize regularly**: Use `/speckit.optimize` to maintain optimal performance
3. **Verify changes**: Use `/speckit.verify` after making changes
